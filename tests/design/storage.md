# 存储层测试设计

> 本文档描述存储层的测试策略和用例设计。
> 版本：v0.1.0-draft | 最后更新：2026-04-22

---

## 1. 测试范围

存储层包含以下子模块：
- 向量存储（Vector Store）
- 稀疏索引（Sparse Store）
- 图存储（Graph Store）
- 元数据存储（Metadata Store）

---

## 2. 向量存储测试

### 2.1 功能测试

| 测试用例 | 操作 | 验证点 |
|----------|------|--------|
| 写入向量 | 插入 1000 个代码 chunk 向量 | 写入成功、无丢失 |
| 向量搜索 | 查询最相似的 top-10 | 返回正确数量、按相似度排序 |
| 批量写入 | 一次性写入 10000 个向量 | 无内存溢出、写入成功 |
| 删除向量 | 删除指定 ID 的向量 | 删除后搜索不再返回 |
| 更新向量 | 修改已有向量的元数据 | 更新后搜索返回最新元数据 |

### 2.2 多后端兼容性测试

| 后端 | 测试内容 |
|------|----------|
| Qdrant | 完整 CRUD + 搜索 |
| Chroma | 完整 CRUD + 搜索 |
| Milvus | 完整 CRUD + 搜索 |

## 3. 稀疏索引测试

| 测试用例 | 操作 | 验证点 |

## 5. 元数据存储测试

| 测试用例 | 操作 | 验证点 |

## 6. 数据一致性测试

| 测试用例 | 场景 | 验证点 |
|----------|------|--------|
| 索引-源码一致性 | 索引后修改源码 | 查询时检测到过期并提示 |
| 行号漂移检测 | 索引后应用 patch | 抽查行号不匹配时报告 |
| 因果链闭环 | Fixes: 指向不存在的 commit | 报告断链 |
| 多版本共存 | v6.12 和 v6.13 同时存在 | 互不干扰 |

---

## 7. 性能测试

| 测试用例 | 目标 |
|----------|------|
| 向量写入吞吐 | ≥ 1000 向量/秒 |
| 向量查询延迟 | P95 ≤ 100ms |
| 图遍历延迟 | ≤ 200ms（深度 5） |
| 存储占用 | 基线 ≤ 2.5GB |
